## 4 · Pla TDD millorat per al sprint d'alfa

L'estratègia: convertir la creació de projecte amb IA en un workflow
**veritablement TDD** · cada etapa té test abans que codi · errors fan
caure el flux ràpid amb missatge accionable.

### 4.1 Etapes del flux IA · una suite per etapa

```
/create
  └─ Etapa 1 · brief input (form)
       └─ test: validateBrief({ name, problem, target }) → {ok, errors}
  └─ Etapa 2 · SOC outline (IA cheap · 500 tokens)
       └─ test: generateSocOutline(brief) → { sections[], cost, ms }
       └─ edge: timeout 10s · IA retorna text no-JSON · API key invàlida
  └─ Etapa 3 · SOP per item (IA · 300-800 tokens cada)
       └─ test: generateSopForItem(socItem) → { steps[], cost, ms }
       └─ edge: 1 item falla · els altres han de continuar (Promise.allSettled)
  └─ Etapa 4 · WOs derivades
       └─ test: deriveWosFromSops(sops) → [{ wo }, ...] · respecta assignee_kind
  └─ Etapa 5 · Persistència KB
       └─ test: persistProject(brief, soc, sops, wos) → projectId
       └─ edge: KB ple · IndexedDB quota error → rollback
  └─ Etapa 6 · Redirect /hub/{id}
       └─ test: navega · primer render no peta
```

**Output esperat:** `js/tests/iaProjectFlowIntegration.test.js` ·
≥30 asserts · 1 happy + ≥6 edge cases.

### 4.2 Fixtures · IA mockejada
- Crear `js/tests/_fixtures/iaResponses.js` amb 5 respostes canòniques
  (cas castellers · cas SaaS · cas event · cas formació · cas dolent).
- Cada fixture té un `expectedSops` perquè el test sigui determinista.
- En CI mai cridem IA real · sempre mockejada via `AiClient.mock(fixtures)`.

### 4.3 Bucle de millora setmanal
- Diumenge nit · cron simulat · agafa les últimes 7 sessions de creació,
  mesura `costTotal` · `latencyP95` · `okRate` per etapa.
- Si una etapa baixa de 90% ok · obre WO automàtic al backlog.
- Dashboard a `/sprint` (post-merge) amb gràfic d'evolució.

### 4.4 Cost · qualitat · ètic
- `aiBudgetService` ja existeix · cal cap d'usuari (max EUR/dia).
- Cada etapa fa `await budget.canSpend(estimateEur)` abans de cridar IA ·
  si supera · degrada graciosament (SOP plantilla en lloc de IA).
- Test específic: `budgetExhausted → fallback ok` (cap excepció · cap cost).

### 4.5 Smoke E2E pre-deploy (`scripts/smoke-create.sh`)
1. `node js/tests/iaProjectFlowIntegration.test.js`
2. Si verd · `node scripts/simulate-create.js` (headless Chrome · cas
   castellers · navega `/create` · espera `/hub/{id}` · verifica DOM).
3. Si verd · publish del bundle.
